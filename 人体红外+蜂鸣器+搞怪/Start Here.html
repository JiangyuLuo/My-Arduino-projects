What to do with your Circuito?

Welcome maker!

This is your project circuito - automatically generated schematic and code for the parts you selected.
We are still in Alpha stages of our technology. We hope you enjoy the service and give us any feedback you may have to make sure we keep improving!

You'll find two folders in this reply:

  • Firmware - includes all the code you need to operate the i/o components using the micro-controller. firmware.ino is the main logic file, but to operate the code you'll need all the files.
  • Diagram - where you'll find the schematic diagram as a Fritzing file. (more info on Fritzing in step 3).

Let's see what else we have here:

First, make sure you have all the parts:

Note that the list contains helper parts like: resistors, capacitors, transistors, etc.. that are essential to the circuit.

Notes before you start

  • If you chose a battery or USB, we may have replaced it with a 12V power source, as some parts can't work otherwise.
  • Some parts may require header soldering in order to connect to other parts.
  • If you're using a component different than the one in Circuito.io, the code and schematic may not work exactly as is, and you'll need to make adjustments.
  • Enjoy your making process :)

Now, assemble the circuit using the Fritzing file:

To use Fritzing files, download the software here.

After download, open the .fzz file located in the Diagram folder

A few things to pay attention to when using the diagram:

  • Our automatically generated schematic drawing is still a work in progress. You many need to move the parts around in the fritzing file to get a better view.
  • Position the parts like in the diagram
  • Make sure you connect the +/- correctly
  • If you need more information on how to use a breadboard - here is a great guide

Let's code!

To upload the code to your controller, download Arduino IDE version 1.6.7 or higher. You can use the Getting Started Guide to set up your work environment.

  1. Open the code folder and double click (don't drag and drop) firmware.ino - all the files in the folder should open in the Arduino development environment. (They're all libraries you'll need to run the code).
  2. Upload to Arduino and Go! The code provided is a sample for you to get started.
